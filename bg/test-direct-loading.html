Perbandingan: Redirect vs Direct Loading

Aspek Sistem Redirect Sistem Direct Loading
URL di Browser Berubah ke localhost:8081 Tetap /smartbk/guru
User Experience User melihat port number User tidak melihat port
Navigation Browser redirect, URL berubah Konten dimuat langsung
URL Sharing User share URL dengan port User share URL bersih
Implementation Browser redirect 301/302 Server-side proxy

Cara Kerja Direct Loading

Flow Process:
  1. User mengakses /smartbk/guru
  2. .htaccess mengarahkan ke app-loader.php?path=guru
  3. app-loader.php membaca file app-guru/.env
  4. Mengambil nilai app.baseURL (contoh: localhost:8081)
  5. Melakukan proxy request ke server aplikasi
  6. Mengembalikan konten langsung ke browser
  7. URL tetap /smartbk/guru di browser

Keuntungan Direct Loading

User Experience
  • URL tetap bersih tanpa port
  • Tidak ada redirect di browser
  • Konsisten dengan domain utama
Security & SEO
  • Port internal tidak terekspos
  • URL structure yang SEO-friendly
  • Centralized access control

Konfigurasi Dynamic

app-guru/.env
app.baseURL = 'http://localhost:8081/'

Proxy ke port 8081

app-ortu/.env
app.baseURL = 'http://localhost:8080/'

Proxy ke port 8080

Custom Configuration
app.baseURL = 'http://localhost:9000/'

Flexible port assignment
