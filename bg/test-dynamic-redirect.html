Perbandingan: Static vs Dynamic

Aspek Sebelum (Static) Sekarang (Dynamic)
URL Target Hardcoded port (8080, 8081, 8082) Baca dari file .env aplikasi
Fleksibilitas Harus edit .htaccess jika ganti port Edit .env aplikasi saja
Server Check Tidak ada validasi Cek status server sebelum redirect
Error Handling Browser error jika server mati Warning & instruksi start server
User Experience Langsung redirect tanpa info Loading screen + status info

Contoh Konfigurasi Dynamic

app-guru/.env
app.baseURL = 'http://localhost:8081/'

Guru akan redirect ke port 8081

app-ortu/.env
app.baseURL = 'http://localhost:8080/'

Ortu akan redirect ke port 8080

Custom Port Example
app.baseURL = 'http://localhost:9000/'

Bisa ganti port sesuai kebutuhan

Keuntungan Sistem Dynamic

Fleksibilitas
  • Ganti port tanpa edit .htaccess
  • Konfigurasi per aplikasi
  • Support development & production
Reliability
  • Server status checking
  • Error handling yang proper
  • User-friendly messages

Cara Kerja Sistem

Flow Process:
  1. User mengakses /smartbk/guru
  2. .htaccess mengarahkan ke redirect.php?path=guru
  3. redirect.php membaca file app-guru/.env
  4. Mengambil nilai app.baseURL
  5. Cek apakah server berjalan di URL tersebut
  6. Jika OK → redirect otomatis, jika tidak → tampilkan error

Testing Commands

Test dengan port berbeda:
# Edit app-guru/.env
app.baseURL = 'http://localhost:9001/'

# Start server di port baru
cd app-guru
php spark serve --port=9001
